# Digital Avatar Live - Requirements Spec V7

## 1. 文档目的
本文件是下一步开发的需求基线，重点提供按钮级需求定义，覆盖功能范围、交互行为、错误处理、验收口径。

## 2. 范围与约束
1. 首发平台：抖音。
2. 单机并发：1 路会话优先。
3. 数字人资产：首版仅导入，不做训练。
4. 音频输入：桌面音频回采优先。
5. 云边边界：LLM/TTS 云端，唇形推理本地 GPU。
6. 最低硬件：RTX 3060。
7. 默认引擎：Wav2Lip。
8. P1 功能：OBS 去重、虚拟摄像头。
9. 回复模式：半自动，命中敏感词需人工确认。

## 3. 需求分层
| 需求域 | 说明 | 优先级 | 核心验收指标 |
| --- | --- | --- | --- |
| R1 实时直播闭环 | 直播消息到数字人播报 | P0 | 首帧 <= 500ms，finalfps >= 25 |
| R2 运营提效 | 话术、策略、账号管理 | P0 | 关键按钮成功率 >= 99% |
| R3 配置与模型 | 引擎、参数、模型生命周期 | P0 | 配置保存可回滚，模型切换 <= 3s |
| R4 可观测与恢复 | 指标、告警、自动恢复 | P0 | 8 小时稳定运行，恢复成功率 >= 99% |
| R5 增强能力 | OBS 去重、虚拟摄像头 | P1 | 不影响 P0 性能指标 |

## 5. 非功能需求
1. 性能
- 端到端首帧 <= 500ms。
- inferfps 与 finalfps 均 >= 25。

2. 稳定性
- 连续运行 8 小时不崩溃。
- 自动恢复成功率 >= 99%。

3. 安全与审计
- 所有按钮级关键操作落审计日志。
- 敏感词命中必须转待审。

4. 可观测
- 最少包含 latency、inferfps、finalfps、queue_usage_ratio、queue_drop_count。

## 6. 验收清单
1. P0 页面按钮全部可操作，且有成功与失败反馈。
2. 高风险文本必须进入待审，不允许自动放行。
3. 模型切换失败可自动回滚。
4. 推流失败可触发自动恢复与人工介入入口。
5. 字段校验与错误码一致，前后端口径统一。

## 7. 本文档用途
1. 用于研发拆解任务（按钮 -> action -> 接口）。
2. 用于测试编写用例（前置条件、成功路径、失败路径）。
3. 用于产品验收（指标口径一致）。
